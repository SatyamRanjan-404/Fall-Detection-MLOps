1. Did we tackle Class Imbalance? (The Dynamic Stride Technique)
Yes, we absolutely tackled it, and Dynamic Stride was the weapon we used.

You had a slight misconception: the dynamic technique does not change the window size. The window size is strictly locked at 50 frames. It changes the step distance (the stride) between each window.

Think of it like taking photographs of an event:

Normal Data (Walking/Sitting): Nothing interesting is happening. We take a "photo" (a 50-frame window) and then step forward by 25 frames to take the next one. This creates a standard 50% overlap.

Fall Data (The Anomaly): This happens in a fraction of a second. As soon as the algorithm sees a fall in the CSV, it switches to "Burst Mode." It takes a 50-frame window, but then steps forward by only 5 frames to take the next one. This creates a massive 90% overlap.

How this fixes Class Imbalance:
Instead of having 10,000 normal windows and only 100 fall windows (which would cause the AI to ignore falls entirely), the burst-mode generates hundreds of slightly shifted fall windows. It mathematically multiplies the minority class without synthesizing fake data.
